# AD9833 DDS Signal Generator (STM32F103C8T6)

A DDS (Direct Digital Synthesis) signal generator based on the STM32F103C8T6 microcontroller and the AD9833 waveform generator chip, with an SPI LCD display and key-based user interface.

## Features

- **Waveform Output**: Sine wave, triangle wave, and square wave generation via AD9833
- **Adjustable Frequency**: Real-time frequency control using onboard keys
- **LCD Display**: 240×280 SPI LCD for displaying frequency, waveform info, and UI elements
- **Chinese Character Support**: LCD driver supports displaying Chinese characters (12×12, 16×16, 24×24, 32×32)
- **CMake Build System**: Modern CMake-based project, generated by STM32CubeMX
- **DMA-driven SPI**: SPI1 with DMA for efficient LCD communication

## Hardware

| Component        | Description                          |
|------------------|--------------------------------------|
| **MCU**          | STM32F103C8T6 (LQFP48, 72 MHz)      |
| **DDS Chip**     | AD9833 (25 MHz external oscillator)  |
| **Display**      | 240×280 SPI TFT LCD                 |
| **Keys**         | 4 push buttons (KEY1–KEY4)          |
| **Crystal**      | 8 MHz HSE, PLL ×9 → 72 MHz SYSCLK  |

## Pin Connections

### AD9833 (Bit-banged SPI via GPIOB)

| Signal | Pin  | Description        |
|--------|------|--------------------|
| FSYNC  | PB15 | Frame sync / CS    |
| SCLK   | PB14 | Serial clock       |
| SDATA  | PB13 | Serial data        |

### LCD (SPI1 + GPIO Control)

| Signal | Pin  | Description        |
|--------|------|--------------------|
| SCK    | PB3  | SPI1 clock         |
| MOSI   | PB5  | SPI1 data          |
| CS     | PA11 | Chip select        |
| DC     | PA12 | Data/Command       |
| RES    | PA15 | Reset              |
| BLK    | PA10 | Backlight control  |

### Keys (Active Low with Pull-up)

| Key   | Pin  | Function          |
|-------|------|--------------------|
| KEY1  | PA5  | (User defined)     |
| KEY2  | PA6  | Frequency down     |
| KEY3  | PA7  | Frequency up       |
| KEY4  | PB0  | (User defined)     |

## Build

### Prerequisites

- **ARM GCC Toolchain** (`arm-none-eabi-gcc`)
- **CMake** ≥ 3.22
- **STM32CubeMX** (optional, for re-generating HAL code)
- **OpenOCD** (for flashing/debugging)

### Build Instructions

```bash
# Configure
cmake -B build -DCMAKE_BUILD_TYPE=Debug

# Build
cmake --build build
```

### Flash

```bash
# Using OpenOCD (config provided)
openocd -f openocd.cfg -c "program build/AD9833.elf verify reset exit"
```

## Usage

1. Power on the device — the LCD initializes and displays the UI
2. The AD9833 outputs a default 10 kHz sine wave on startup
3. **KEY3** (PA7): Increase frequency (+1 kHz step)
4. **KEY2** (PA6): Decrease frequency (-1 kHz step)
5. The LCD updates to show the current frequency in real-time

### Waveform Selection

The AD9833 driver supports three waveform modes (defined in `ad9833.h`):

- `SIN_WAVE` (1) — Sine wave output
- `TRI_WAVE` (0) — Triangle wave output
- `SQU_WAVE` (2) — Square wave output

## AD9833 Driver API

```c
// Initialize AD9833 with default settings (1 kHz sine on FREQ0)
void ad9833_init(void);

// Configure waveform output
//  Freq      — Frequency in Hz
//  Freq_SFR  — Frequency register select (0 or 1)
//  WaveMode  — SIN_WAVE / TRI_WAVE / SQU_WAVE
//  Phase     — Phase offset value
void AD9833_WaveSeting(double Freq, unsigned int Freq_SFR,
                       unsigned int WaveMode, unsigned int Phase);

// Quick waveform switch helpers
void wave_out(void);           // FREQ1 → 2 kHz default
void triangle_wave_out(void);  // FREQ0 → triangle wave
void square_wave_out(void);    // FREQ0 → square wave
```
